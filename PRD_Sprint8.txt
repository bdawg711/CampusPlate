  meal plans around their class schedule. No OAuth needed — Canvas
  lets every student export their calendar as an iCal feed URL from
  Canvas Settings → Calendar.

  TASK 4.1 — Canvas setup flow in Settings
  -------------------------------------------------------
  In the Settings screen, add a new section: "Integrations"

  If not premium:
    - Show "Canvas Calendar" row with lock icon
    - Tapping opens PaywallModal

  If premium:
    - Show "Canvas Calendar" row with status (Connected / Not Connected)
    - Tapping opens a modal/screen:
      - Header: "Connect Canvas Calendar"
      - Instructions (step by step):
        1. "Open Canvas in your browser"
        2. "Go to Account → Settings"
        3. "Scroll to Calendar → Click Calendar Feed"
        4. "Copy the URL that appears"
      - TextInput for pasting the iCal URL
        - Placeholder: "Paste your Canvas calendar URL here"
        - Validate: must start with "https://" and contain ".ics" or
          "calendar" in the URL
      - "Connect" button
      - If already connected: show "Connected ✓" with option to
        disconnect or update URL

  On connect:
    - Fetch the iCal URL to validate it returns valid iCalendar data
    - Parse with a simple iCal parser to extract events
    - Store the URL in Supabase profiles table:
      ALTER TABLE profiles ADD COLUMN IF NOT EXISTS
        canvas_ical_url TEXT;
    - Show success state

  DONE WHEN: Premium users can paste iCal URL, it validates and
  stores in their profile.

  TASK 4.2 — iCal parser utility
  -------------------------------------------------------
  Create src/utils/calendar.ts:

    - fetchAndParseCalendar(icalUrl: string): CalendarEvent[]
    - Parse iCalendar format (VEVENT blocks):
      - SUMMARY → event name (class name)
      - DTSTART / DTEND → start/end times
      - RRULE → recurrence (weekly classes)
      - LOCATION → building/room
    - Filter to only include events for the current week
    - Return sorted array of events with:
      { name, dayOfWeek, startTime, endTime, location }
    - Handle timezone conversion (Canvas uses UTC, convert to ET)

  DONE WHEN: Given an iCal URL, returns parsed class schedule.

FEATURE 5 — AI MEAL PLANNING (PREMIUM)
================================================================================

  PURPOSE:
  Use the AI to generate a full day's meal plan based on the user's
  nutrition goals, today's dining hall menus, and (if connected)
  their class schedule. This is the killer premium feature.

  TASK 5.1 — Meal Plan generation via Edge Function
  -------------------------------------------------------
  Add a new mode to the Edge Function: mode: "meal_plan"

  Input:
    {
      mode: "meal_plan",
      schedule: [
        { name: "CS 2104", start: "09:30", end: "10:45", location: "McBryde" },
        { name: "COMM 1016", start: "14:00", end: "15:15", location: "Shanks" }
      ],  // optional, empty array if no Canvas connected
      goals: { calories: 2500, protein: 100, carbs: 100, fat: 94 },
      preferences: { dietary: ["vegetarian"], allergies: [] }
    }

  System prompt:
    "You are a campus meal planning assistant for Virginia Tech.
     Generate a complete daily meal plan using ONLY items from today's
     dining hall menus (provided below). The plan must:
     1. Fit within the user's calorie and macro goals
     2. Schedule meals around their class times (if provided)
     3. Respect dietary preferences and allergies
     4. Suggest specific dining halls and items
     5. Include estimated nutrition totals per meal and daily total

     Respond with ONLY a JSON object:
     {
       meals: [
         {
           type: 'Breakfast' | 'Lunch' | 'Dinner' | 'Snack',
           time: '8:00 AM',
           location: 'D2',
           items: [{ name: 'Scrambled Eggs', calories: 180, protein: 12, carbs: 2, fat: 14 }],
           totalCalories: 450,
           note: 'Before your 9:30 CS class'
         }
       ],
       dailyTotal: { calories: 2480, protein: 98, carbs: 105, fat: 90 },
       tip: 'You have a 2-hour gap at noon — perfect time for a bigger lunch at Owen's.'
     }"

  Rate limit: 3 meal plans per day (tracked in ai_usage as plan_count)
  Add column: ALTER TABLE ai_usage ADD COLUMN IF NOT EXISTS plan_count INTEGER DEFAULT 0;

  DONE WHEN: Edge Function generates structured meal plans from menus.

  TASK 5.2 — Meal Plan UI
  -------------------------------------------------------
  Create src/components/MealPlanView.tsx:

  Accessible from:
    - A "Plan My Day" button on the Dashboard (prominent, below greeting)
    - Only visible to premium users

  Layout:
    - Header: "Today's Meal Plan" with date
    - If user has Canvas connected, show schedule context:
      "Built around your 3 classes today"
    - Timeline of meals, each as a glass card:
      - Meal type pill (Breakfast/Lunch/Dinner/Snack)
      - Suggested time + dining hall
      - List of items with calories
      - Note from AI (schedule context)
      - "Log These Items" button → logs all items in one tap
    - Daily total summary card at bottom:
      - Calories, protein, carbs, fat vs goals
      - Ring visualization showing how the plan fits goals
    - "Regenerate" button (uses 1 of 3 daily plans)
    - Remaining plans count: "2 plans remaining today"

  Loading state:
    - Pulsing shimmer cards while AI generates

  Error state:
    - "Couldn't generate plan. Try again." with retry button

  DONE WHEN: Premium users can generate and view daily meal plans,
  log meals from the plan in one tap.

FEATURE 7 — WATER BOTTLE REDESIGN (DEER PARK STYLE)
================================================================================

  PURPOSE:
  The current water bottle SVG is too narrow/skinny and doesn't look
  good. Redesign it to look like a Deer Park water bottle shape —
  wider body, tapered shoulder, narrow neck, small cap.

  TASK 7.1 — Redesign WaterTracker SVG
  -------------------------------------------------------
  In src/components/WaterTracker.tsx:

  New bottle shape (approximate Deer Park proportions):
    - Small rectangular cap at top (narrow, ~20% of body width)
    - Short narrow neck below cap (~25% of body width)
    - Tapered shoulder widening from neck to body
    - Wide cylindrical body (main area, ~60% of total height)
    - Slight taper at bottom, flat base

  SVG path approach — use a single path for the bottle outline:
    viewBox="0 0 60 100"

    Cap:    rectangle, x: 22-38, y: 0-6
    Neck:   narrow, x: 20-40, y: 6-15
    Shoulder: curves widening, y: 15-25
    Body:   wide rectangle, x: 8-52, y: 25-90
    Base:   flat, slightly rounded corners, y: 90-95

  Fill animation:
    - Use a clipPath matching the bottle shape
    - Inside the clip: a rectangle that grows from bottom to top
    - At 0%: rect top is at the bottom (no fill visible)
    - At 100%: rect top is at y=0 (entire bottle filled, including
      neck and cap — the FULL bottle is blue)
    - Fill color: colors.accent or a nice water blue
    - Animated with React Native Animated or Reanimated

  Keep existing functionality:
    - Tap to add water (same increment)
    - Display oz count and goal
    - Same position on Dashboard

  DONE WHEN: Water bottle looks like a Deer Park bottle, fills
  completely from bottom to top including through the neck and cap.

GLOBAL RULES (same as previous sprints)
================================================================================

  DATES:
    const d = new Date();
    const today = `${d.getFullYear()}-${String(d.getMonth()+1).padStart(2,'0')}-${String(d.getDate()).padStart(2,'0')}`;
    NEVER use toISOString() for dates.

  COLORS:
    Always use useTheme() → colors object. Never hardcode hex values.
    Glass cards: colors.cardGlass bg + colors.cardGlassBorder border.

  FONTS:
    Headers: Outfit_700Bold or Outfit_800ExtraBold
    Body: DMSans_400Regular, DMSans_500Medium, DMSans_600SemiBold
    Never use fontWeight with fontFamily — pick the correct font file.

  ICONS:
    Feather icons from @expo/vector-icons only. No emojis for UI chrome.

  SUPABASE:
    All queries wrapped in try/catch.
    Always filter by user_id.
    nutrition joins return arrays: item.nutrition[0].calories
    weight_logs column: weight_lbs (not weight)

  ERRORS:
    User-friendly alerts. Never show raw error messages.
    Console.warn for non-critical failures.

  TAB BAR:
    All screens need paddingBottom: 100 for FloatingTabBar clearance.
