## Feature 1: Favorites / Bookmarks

**What:** Students tap a heart icon on any menu item to save it. A "Favorites" section shows all saved items. When a favorited item appears on today's menu at any hall, it surfaces in "For You" on the dashboard.

**Why rec_num:** The same dish gets a new `menu_item_id` every day at every hall, but keeps the same `rec_num` in FoodPro. Favoriting by `rec_num` means "Grilled Chicken" is one favorite regardless of which hall or date it appears on.

### Database

```sql
-- Run in Supabase SQL Editor
CREATE TABLE favorites (
  id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  rec_num text NOT NULL,
  item_name text NOT NULL,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE favorites ADD CONSTRAINT favorites_user_rec_unique UNIQUE (user_id, rec_num);
CREATE INDEX idx_favorites_user ON favorites(user_id);
```

## Feature 2: Dining Hall Ratings

**What:** Students rate dining halls 1-5 stars (like a Google review). Optional short text review. Average rating shows on the hall card in browse. Recent reviews visible when you tap into a hall.

### Database

```sql
-- Run in Supabase SQL Editor
CREATE TABLE hall_ratings (
  id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  dining_hall_id bigint NOT NULL REFERENCES dining_halls(id) ON DELETE CASCADE,
  rating smallint NOT NULL CHECK (rating >= 1 AND rating <= 5),
  review_text text,
  date date NOT NULL,
  created_at timestamptz DEFAULT now(),
  updated_at timestamptz DEFAULT now()
);

ALTER TABLE hall_ratings ADD CONSTRAINT hall_ratings_user_hall_date_unique UNIQUE (user_id, dining_hall_id, date);
CREATE INDEX idx_hall_ratings_hall ON hall_ratings(dining_hall_id);
CREATE INDEX idx_hall_ratings_user ON hall_ratings(user_id);

-- RPC to get average rating per hall
CREATE OR REPLACE FUNCTION get_hall_averages()
RETURNS TABLE(dining_hall_id bigint, avg_rating numeric, total_ratings bigint)
LANGUAGE sql STABLE
AS $$
  SELECT dining_hall_id, ROUND(AVG(rating)::numeric, 1), COUNT(*)
  FROM hall_ratings
  GROUP BY dining_hall_id;
$$;

-- RPC to get recent reviews for a hall
CREATE OR REPLACE FUNCTION get_hall_reviews(p_hall_id bigint, p_limit int DEFAULT 20)
RETURNS TABLE(rating smallint, review_text text, date date, user_name text)
LANGUAGE sql STABLE
AS $$
  SELECT hr.rating, hr.review_text, hr.date, p.name
  FROM hall_ratings hr
  LEFT JOIN profiles p ON hr.user_id = p.id
  WHERE hr.dining_hall_id = p_hall_id
  AND hr.review_text IS NOT NULL
  ORDER BY hr.created_at DESC
  LIMIT p_limit;
$$;
```

## Feature 3: What's Open Right Now

**What:** On the dashboard and browse tab, show which dining halls are currently open based on hardcoded operating hours. Halls that are open get a green "Open" badge; closed halls get a gray "Closed" badge with next opening time.

### Database

```sql
-- Run in Supabase SQL Editor
CREATE TABLE dining_hall_hours (
  id bigint GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  dining_hall_id bigint NOT NULL REFERENCES dining_halls(id) ON DELETE CASCADE,
  day_of_week smallint NOT NULL CHECK (day_of_week >= 0 AND day_of_week <= 6),
  meal text NOT NULL,
  open_time time NOT NULL,
  close_time time NOT NULL,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE dining_hall_hours ADD CONSTRAINT dining_hall_hours_unique UNIQUE (dining_hall_id, day_of_week, meal);
CREATE INDEX idx_hall_hours_hall_day ON dining_hall_hours(dining_hall_id, day_of_week);
```

**day_of_week:** 0 = Sunday, 1 = Monday, ..., 6 = Saturday (matches JavaScript `getDay()`).

## Feature 5: Smart Recommendations (For You)

**What:** The "For You" section on the dashboard becomes intelligent. Instead of static content, it shows personalized collections based on the student's goals, history, favorites, and time of day.

### No New Tables Needed

This feature queries existing tables: `profiles` (goals), `meal_logs` (history), `favorites`, `menu_items` + `nutrition`, `hall_ratings`.

### For You Collections

These are horizontal scroll rows on the dashboard. Show whichever ones have data, in this order:

1. **"Your Favorites Today"** — Favorited items that are on today's menu (from Feature 1).
2. **"Fits Your Macros"** — Items from today's menu where: if user goal is `bulk`, show items with protein_g > 25. If goal is `lose`, show items with calories < 400. If goal is `maintain`, show balanced items (calories 300-600, protein > 15g). Filter to the current meal period. Max 10 items.
3. **"Top Rated Halls"** — Halls sorted by average rating (from Feature 2), with their current open status (from Feature 3). Max 5 halls.
4. **"Try Something New"** — Items on today's menu at the user's home hall that they have NOT logged before (compare rec_nums in meal_logs vs today's menu_items). Max 8 items.
5. **"Quick & Light"** — Items under 300 calories from any currently-open hall. Good for snack runs.

CREATE POLICY "Users can insert own favorites" ON favorites FOR INSERT WITH CHECK (auth.uid() = user_id);
