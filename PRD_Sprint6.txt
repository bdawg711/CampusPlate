FEATURE 1 — CUSTOM MEAL LOGGING
================================================================================

  PURPOSE:
  Users eat off-campus (Chipotle, home-cooked, snacks, protein shakes, etc.)
  and need to log those meals to get an accurate daily picture. Right now
  logging only works for VT dining hall items. This feature adds a manual
  entry flow accessible from the Dashboard and AI tab.

  TASK 1.1 — Create custom_meals table in Supabase
  -------------------------------------------------------
  SQL:
    CREATE TABLE custom_meals (
      id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
      user_id UUID REFERENCES auth.users(id) NOT NULL,
      name TEXT NOT NULL,
      calories INTEGER NOT NULL DEFAULT 0,
      protein_g REAL NOT NULL DEFAULT 0,
      total_carbs_g REAL NOT NULL DEFAULT 0,
      total_fat_g REAL NOT NULL DEFAULT 0,
      date TEXT NOT NULL,
      meal_period TEXT NOT NULL DEFAULT 'Snack',
      created_at TIMESTAMPTZ DEFAULT NOW()
    );

    -- Enable RLS
    ALTER TABLE custom_meals ENABLE ROW LEVEL SECURITY;

    -- Users can only see/modify their own entries
    CREATE POLICY "Users can view own custom meals"
      ON custom_meals FOR SELECT USING (auth.uid() = user_id);
    CREATE POLICY "Users can insert own custom meals"
      ON custom_meals FOR INSERT WITH CHECK (auth.uid() = user_id);
    CREATE POLICY "Users can update own custom meals"
      ON custom_meals FOR UPDATE USING (auth.uid() = user_id);
    CREATE POLICY "Users can delete own custom meals"
      ON custom_meals FOR DELETE USING (auth.uid() = user_id);

  DONE WHEN: Table exists, RLS policies active. Verify in Supabase dashboard.
